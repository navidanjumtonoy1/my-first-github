GIT, GITHUB & HOSTING CHEAT SHEET

এই Cheat Sheet কীভাবে ব্যবহার করবে?

প্রতিটি card-এ আছে real coding need, short command/example, সহজ Bangla explanation, beginner note এবং common mistake। Main topics শেষ হলে আছে VS Code/Terminal/Git shortcuts, mistakes vs best practices এবং viva Q&A।

1

What is Version Control?

Why needed: Project code-এর change history track করা।

Command / Example

Version Control = change tracking system

Explanation: Version control code-এর previous version save করে, তাই ভুল হলে আগের অবস্থায় ফেরা যায়।

💡 Note: একজন developer হলেও version control use করা professional habit।

⚠️ Mistake: Only final file save করে old versions হারিয়ে ফেলা।

2

What is Git?

Why needed: Local computer-এ code history manage করা।

Command / Example

git --version

Explanation: Git হলো distributed version control system। এটি code changes track করে।

💡 Note: Git তোমার computer-এ কাজ করে, internet লাগবেই এমন না।

⚠️ Mistake: Git install না করে GitHub use করতে চাওয়া।

3

What is GitHub?

Why needed: Online-এ repository store, share এবং collaborate করা।

Command / Example

https://github.com

Explanation: GitHub হলো Git repositories host করার online platform।

💡 Note: Portfolio project রাখতে GitHub খুব important।

⚠️ Mistake: Git আর GitHub একই জিনিস ভাবা।

4

Difference between Git and GitHub

Why needed: Tool আর online platform-এর পার্থক্য বুঝতে।

Command / Example

Git = local tool
GitHub = online platform

Explanation: Git code history manage করে, GitHub সেই Git repository online রাখে।

💡 Note: Git ছাড়া GitHub workflow ভালোভাবে বোঝা কঠিন।

⚠️ Mistake: GitHub account থাকলেই Git শেখা হয়ে গেছে ভাবা।

5

Source Control Meaning

Why needed: Code changes organize এবং manage করা।

Command / Example

Source Control = managing code changes

Explanation: Source control মানে source code-এর changes track, compare, save এবং share করা।

💡 Note: VS Code Source Control panel Git কাজ সহজ করে।

⚠️ Mistake: Source control ছাড়া বড় project manage করা।

6

Installing Git

Why needed: Git command use করার জন্য Git install দরকার।

Command / Example

Download Git from official website
Then install it

Explanation: Git install করলে terminal-এ git command চালানো যায়।

💡 Note: Install শেষে terminal restart করো।

⚠️ Mistake: Install করার পর git --version check না করা।

7

Checking Git Version

Why needed: Git properly install হয়েছে কিনা check করা।

Command / Example

git --version

Explanation: এই command installed Git version দেখায়।

💡 Note: প্রথম Git setup-এর পর এই command run করো।

⚠️ Mistake: Command না চললে Git PATH issue হতে পারে।

8

Git Configuration

Why needed: Commit history-তে author name/email show করা।

Command / Example

git config --global user.name "Eng. P"
git config --global user.email "you@example.com"

Explanation: Git config তোমার commit author information set করে।

💡 Note: GitHub email-এর সাথে মিল থাকলে ভালো।

⚠️ Mistake: Wrong email দিয়ে commit করা।

9

Repository Meaning

Why needed: Project folder-এর Git-tracked version।

Command / Example

Repository = project + Git history

Explanation: Repository হলো এমন project folder যেখানে Git changes track করে।

💡 Note: প্রতিটি project-এর জন্য আলাদা repo রাখো।

⚠️ Mistake: সব project এক repo-তে মিশিয়ে ফেলা।

10

Local vs Remote Repository

Why needed: Computer আর online repository-এর difference বোঝা।

Command / Example

Local repo = your computer
Remote repo = GitHub/online

Explanation: Local repo তোমার PC-তে থাকে, remote repo GitHub-এর মতো online platform-এ থাকে।

💡 Note: Push করলে local changes remote-এ যায়।

⚠️ Mistake: Local save করলেই GitHub-এ upload হয়েছে ভাবা।

11

Working Directory, Staging Area, Repository

Why needed: Git flow বোঝার জন্য core concept।

Command / Example

Working directory → git add → staging → git commit → repository

Explanation: Working directory-তে edit হয়, staging-এ selected changes যায়, commit হলে repo history-তে save হয়।

💡 Note: এই flow Git শেখার foundation।

⚠️ Mistake: git add ছাড়া commit করলে অনেক change বাদ পড়ে যায়।

12

git init

Why needed: New project folder-কে Git repository বানানো।

Command / Example

git init

Explanation: এই command current folder-এ hidden .git folder তৈরি করে।

💡 Note: New local project শুরু করলে git init run করো।

⚠️ Mistake: Wrong folder-এ git init চালানো।

13

git status

Why needed: Current project changes দেখা।

Command / Example

git status

Explanation: কোন file modified, staged, untracked তা দেখায়।

💡 Note: প্রতিটি add/commit/push-এর আগে status check করো।

⚠️ Mistake: Status না দেখে ভুল file commit করা।

14

git add

Why needed: Changes staging area-তে পাঠানো।

Command / Example

git add index.html
git add .

Explanation: git add selected file বা সব changes commit-এর জন্য ready করে।

💡 Note: git add . সব current changes stage করে।

⚠️ Mistake: অপ্রয়োজনীয় file stage হয়ে যাওয়া।

15

git commit

Why needed: Project-এর saved snapshot তৈরি করা।

Command / Example

git commit -m "add homepage"

Explanation: Commit হলো project-এর একটি saved version।

💡 Note: ছোট logical commit করো।

⚠️ Mistake: Meaningless message যেমন 'update' বারবার লেখা।

16

Writing Good Commit Messages

Why needed: Future history বুঝতে সহজ করা।

Command / Example

git commit -m "create contact form"

Explanation: Good commit message clearly বলে কী change করা হয়েছে।

💡 Note: Verb দিয়ে শুরু করো: add, fix, update, remove।

⚠️ Mistake: Too vague message লেখা।

17

git log

Why needed: Commit history দেখা।

Command / Example

git log
git log --oneline

Explanation: git log previous commits দেখায়। --oneline short view দেয়।

💡 Note: History quickly দেখতে --oneline useful।

⚠️ Mistake: Long log থেকে বের হতে q চাপতে ভুলে যাওয়া।

18

git diff

Why needed: Commit করার আগে changes compare করা।

Command / Example

git diff

Explanation: git diff unstaged changes line by line দেখায়।

💡 Note: Commit-এর আগে diff check করলে mistake ধরা পড়ে।

⚠️ Mistake: Diff না দেখে unwanted change commit করা।

19

git restore

Why needed: Unwanted local changes discard করা।

Command / Example

git restore index.html

Explanation: File-এর uncommitted changes আগের অবস্থায় ফিরিয়ে নেয়।

💡 Note: Use carefully, changes হারিয়ে যেতে পারে।

⚠️ Mistake: ভুল file restore করে দরকারি কাজ হারানো।

20

git rm

Why needed: File remove করে Git-কে জানানো।

Command / Example

git rm old-file.html

Explanation: File delete করে deletion staging area-তে রাখে।

💡 Note: Git-tracked file remove করতে useful।

⚠️ Mistake: Manual delete করে status check না করা।

21

.gitignore File

Why needed: অপ্রয়োজনীয়/private file Git থেকে বাদ দেওয়া।

Command / Example

node_modules/
.env
.DS_Store

Explanation: .gitignore বলে কোন files/folders Git track করবে না।

💡 Note: Node project-এ node_modules ignore করো।

⚠️ Mistake: Secret key বা .env commit করা।

22

README.md File

Why needed: Project explain করা।

Command / Example

# My Project
Simple HTML CSS website.

Explanation: README.md project-এর description, setup, usage explain করে।

💡 Note: GitHub portfolio-তে README খুব important।

⚠️ Mistake: Empty repository রেখে README না দেওয়া।

Command / Example

⚠️ Mistake: Plain messy text README-তে লেখা।

24

Branch Meaning

Why needed: Main code না ভেঙে আলাদা কাজ করা।

Command / Example

main
feature-navbar

Explanation: Branch হলো project-এর আলাদা line of development।

💡 Note: New feature-এর জন্য branch use করা ভালো।

⚠️ Mistake: সব কাজ main branch-এ করা।

25

git branch

Why needed: Branches list দেখা বা branch তৈরি করা।

Command / Example

git branch
git branch feature-login

Explanation: git branch existing branches দেখায় এবং নতুন branch বানাতে পারে।

💡 Note: Current branch-এর পাশে * থাকে।

⚠️ Mistake: কোন branch-এ আছো না দেখে কাজ করা।

26

git switch / git checkout

Why needed: Branch change করা।

Command / Example

git switch feature-login
git checkout main

Explanation: Branch বদলাতে switch বা checkout use হয়।

💡 Note: Modern Git-এ switch বেশি clear।

⚠️ Mistake: Unsaved changes রেখে branch switch করা।

27

git merge

Why needed: এক branch-এর changes অন্য branch-এ আনা।

Command / Example

git switch main
git merge feature-login

Explanation: Merge feature branch-এর কাজ main branch-এ combine করে।

💡 Note: Merge করার আগে status clean রাখো।

⚠️ Mistake: Conflict না বুঝে merge করা।

28

Merge Conflict Meaning

Why needed: Same file-এর same line-এ conflicting change।

Command / Example

<<<<<<< HEAD
current
=======
incoming
>>>>>>> branch

Explanation: Git automatic merge করতে না পারলে merge conflict হয়।

💡 Note: Conflict marker দেখে manually ঠিক করতে হয়।

⚠️ Mistake: Conflict marker রেখে commit করা।

29

How to Solve Merge Conflict

Why needed: Conflict ঠিক করে final version save করা।

Command / Example

1. Open conflicted file
2. Choose correct code
3. Remove markers
4. git add .
5. git commit

Explanation: Conflict resolve মানে final correct code রেখে conflict markers remove করা।

💡 Note: Slowly read current and incoming changes।

⚠️ Mistake: না বুঝে সব delete করা।

30

git clone

Command / Example

git clone https://github.com/user/project.git

Explanation: Clone GitHub repository তোমার computer-এ download করে।

origin = remote repo nickname

💡 Note: Remote থাকলে push/pull করা যায়।

⚠️ Mistake: Remote connect না করে push করা।

git remote add origin https://github.com/user/repo.git

Explanation: origin নামে remote URL save হয়।

💡 Note: First push-এর আগে remote add করতে হয়।

⚠️ Mistake: Wrong URL add করা।

33

Command / Example

git remote -v

Explanation: fetch এবং push URL দেখায়।

💡 Note: Push problem হলে remote URL check করো।

34

git push

Why needed: Local commits GitHub-এ upload করা।

Command / Example

git push -u origin main
git push

Explanation: Push local commits remote repository-তে পাঠায়।

💡 Note: First push-এ -u origin main helpful।

⚠️ Mistake: Commit ছাড়া push করলে কিছু upload হবে না।

35

git pull

Why needed: Remote latest changes local-এ আনা।

Command / Example

git pull origin main

Explanation: Pull remote changes download করে এবং merge করে।

💡 Note: Team project-এ কাজের আগে pull করো।

⚠️ Mistake: Local changes check না করে pull করা।

36

git fetch

Why needed: Remote updates download কিন্তু merge না করা।

Command / Example

git fetch

Explanation: Fetch remote changes আনে কিন্তু working branch modify করে না।

💡 Note: Safe checking-এর জন্য fetch useful।

⚠️ Mistake: fetch আর pull একই ভাবা।

37

Pull vs Fetch

Why needed: Safe update workflow বুঝা।

Command / Example

fetch = download only
pull = fetch + merge

Explanation: fetch শুধু updates আনে, pull updates এনে local branch-এ merge করে।

💡 Note: Conflict avoid করতে আগে fetch করে দেখতে পারো।

⚠️ Mistake: pull করলে direct merge হয় সেটা না জানা।

⚠️ Mistake: master/main confusion করা।

Git, GitHub and Hosting Viva Questions and Answers

Beginner-level viva/interview preparation — short and easy to memorize

Q1. Version control কী?

Answer: Version control হলো code changes track করার system, যাতে previous version-এ ফিরতে পারি।

Q2. Git কী?

Answer: Git হলো distributed version control system যা local computer-এ code history manage করে।

Q3. GitHub কী?

Answer: GitHub হলো online platform যেখানে Git repositories store, share এবং collaborate করা যায়।

Q4. Git এবং GitHub-এর difference কী?

Answer: Git local version control tool, GitHub online hosting platform for Git repositories।

Q5. Source control কী?

Answer: Source control মানে source code changes manage, track এবং share করা।

Q6. Developers Git কেন use করে?

Answer: Code history, collaboration, backup, branch workflow এবং rollback-এর জন্য Git use করে।

Q7. Repository কী?

Answer: Repository হলো project folder plus Git history।

Q8. Local repository এবং remote repository-এর difference কী?

Answer: Local repo computer-এ থাকে, remote repo GitHub-এর মতো online platform-এ থাকে।

Q9. Working directory কী?

Answer: যেখানে তুমি actual files edit করো সেটি working directory।

Q10. Staging area কী?

Answer: Commit করার আগে selected changes রাখার middle area হলো staging area।

Q11. Commit কী?

Answer: Commit হলো project-এর saved snapshot/version।

Q12. git init কী করে?

Answer: Current folder-কে Git repository বানায়।

Q13. git status কী দেখায়?

Answer: Modified, staged, untracked files এবং current branch status দেখায়।

Q14. git add কী করে?

Answer: File changes staging area-তে রাখে।

Q15. git commit কী করে?

Answer: Staged changes repository history-তে save করে।

Q16. Good commit message কেমন?

Answer: Short, clear এবং action-based: যেমন 'add contact form'।

Q17. git log কী দেখায়?

Answer: Commit history দেখায়।

Q18. git diff কী?

Answer: Commit করার আগে file changes compare করে।

Q19. .gitignore কী?

Answer: যেসব file/folder Git track করবে না তা .gitignore-এ লেখা হয়।

Q20. node_modules push করা উচিত না কেন?

Answer: এটি খুব বড় এবং package manager দিয়ে আবার install করা যায়।

Q21. README.md কী?

Answer: Project description, setup, usage explain করার file।

Q22. Markdown কী?

Answer: Simple text formatting syntax, README সুন্দর করতে use হয়।

Q23. Branch কী?

Answer: Project-এর আলাদা line of development।

Q24. Branch কেন use করি?

Answer: Main code না ভেঙে new feature/experiment করতে।

Q25. main branch এবং feature branch-এর difference কী?

Answer: main stable production code, feature branch new work-এর জন্য।

Q26. git merge কী?

Answer: এক branch-এর changes অন্য branch-এ combine করে।

Q27. Merge conflict কী?

Answer: Git automatic merge করতে না পারলে conflict হয়।

Q28. Merge conflict কীভাবে solve করো?

Answer: Conflict file open করে correct code রাখি, markers remove করি, add এবং commit করি।

Q29. git clone কী?

Answer: Remote repository local computer-এ copy করে।

Q30. remote origin কী?

Answer: origin হলো remote repository-এর default nickname।

Q31. git remote -v কী দেখায়?

Answer: Connected remote URLs দেখায়।

Q32. git push কী?

Answer: Local commits remote repository-তে upload করে।

Q33. git pull কী?

Answer: Remote changes local branch-এ download এবং merge করে।

Q34. git pull এবং git fetch-এর difference কী?

Answer: fetch শুধু download করে, pull fetch + merge করে।

Q35. GitHub repository কী?

Answer: GitHub-এ hosted project repository।

Q36. Project GitHub-এ কীভাবে upload করো?

Answer: git init, add, commit, remote add, push workflow follow করি।

Q37. GitHub Issues কী?

Answer: Bug, task, feature request track করার system।

Q38. Pull Request কী?

Answer: Branch changes review করে merge করার request।

Q39. Fork কী?

Answer: অন্যের repo নিজের GitHub account-এ copy করা।

Q40. GitHub Pages কী?

Answer: Static websites host করার GitHub feature।

Q41. Hosting কী?

Answer: Website files internet-এ accessible করার process।

Q42. Deployment কী?

Answer: Project live server/hosting platform-এ publish করা।

Q43. GitHub Pages, Netlify, Vercel-এর difference কী?

Answer: GitHub Pages simple static site, Netlify static/frontend, Vercel modern frontend/Next.js projects-এর জন্য popular।

Q44. GitHub Pages-এ কী type project host করা যায়?

Answer: Static HTML/CSS/JS website host করা যায়।

Q45. Build folder কী?

Answer: Production-ready generated output folder, যেমন dist বা build।

Q46. Custom domain কী?

Answer: Hosting default URL বাদ দিয়ে নিজের domain use করা।

Q47. Environment variable কী?

Answer: Secret/config value code-এর বাইরে রাখার way।

Q48. SSH এবং HTTPS-এর difference কী?

Answer: HTTPS token-based URL use করে, SSH key-based authentication use করে।

Q49. Personal Access Token কী?

Answer: GitHub authentication-এর জন্য password alternative token।

Q50. VS Code Source Control panel কী?

Answer: VS Code-এর Git GUI panel, যেখানে changes, staging, commit দেখা যায়।
